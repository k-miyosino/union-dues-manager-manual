# 第8章 トラブルシューティング

---

## 8-1. ログファイルの確認

障害調査の第一歩はログの確認です。

**ログファイルの場所：**

```
%LOCALAPPDATA%\UnionDuesCollectionManager\logs\
（例：C:\Users\username\AppData\Local\UnionDuesCollectionManager\logs\）
```

エクスプローラーのアドレスバーに上記のパスを入力すると直接開けます。

- ログファイルはテキスト形式で、日付ごとに作成されます
- エラーが発生した日付のファイルを確認します
- `[ERR]` または `[WRN]` で始まる行がエラー・警告です

### ログの読み方（例）

```
[2025-03-01 09:12:34 ERR] Kintone更新エラー RecordId=123
  System.Exception: Unauthorized
  ...（スタックトレース）...
```

## 8-2. よくある問題と対処法

### Kintone接続・認証

| 症状 | 原因 | 対処 |
|------|------|------|
| 「認証エラー」が表示される | APIトークンが無効または期限切れ | Kintoneでトークンを再発行し、`kintone_api_tokens.json` を更新（→[第3章](03_kintone_config.md)） |
| 「接続できません」 | Kintoneドメインが誤っている、またはネットワーク不通 | `kintone_config.json` のドメインを確認。ブラウザでKintoneにアクセスできるか確認 |
| 「アプリが見つかりません」 | アプリIDが誤っている | `kintone_app_ids.json` のIDをKintoneのURLで確認して修正 |
| データ取得でエラーが続く | APIアクセス権限が不足 | Kintoneのアプリ設定でAPIトークンの権限を確認（→[第3章](03_kintone_config.md)） |

### 起動・データベース

| 症状 | 原因 | 対処 |
|------|------|------|
| 起動時に「データベース初期化エラー」 | DBファイルが破損、またはアクセス権限不足 | DBファイルのパスを確認。別プロセスがロックしていないか確認。解消しない場合はバックアップからリストア（→[第7章](07_backup_migration.md)） |
| 起動時に「マイグレーションエラー」 | スキーマ更新に失敗 | ログを確認。DBのバックアップを取得後、担当開発者に連絡 |
| 口座情報が表示されない（空欄） | 暗号化キーが変わっている、または削除された | バックアップからリストア（→[第7章](07_backup_migration.md)）。バックアップがない場合は口座情報の再登録が必要 |
| PC移行後にデータが表示されない | DBを直接コピーして移行した | バックアップ機能を使ってリストアし直す（→[第7章](07_backup_migration.md)） |

### 締め処理・消込

| 症状 | 原因 | 対処 |
|------|------|------|
| 請求一覧が0件 | KintoneのM_組合員物件所有状況にデータがない、または開始日の設定誤り | Kintone上のデータと開始日・終了日の設定を確認（→運用マニュアル第7章） |
| 「委託先口座に無効な設定があります」 | 委託先口座が無効になっている | アプリ内の委託先口座マスタで「有効」に変更 |
| 消込でKintone更新エラー | 請求レコードの書き込み権限不足 | M_請求レコードのAPIトークンに「編集」権限があるか確認 |

## 8-3. 設定ファイルのリセット手順

設定ファイルが破損した場合のリセット手順です。

### Kintone設定のリセット

1. 以下のファイルを削除します（または内容を空にします）
   ```
   %LOCALAPPDATA%\AccountReconciler\kintone_config.json
   %LOCALAPPDATA%\AccountReconciler\kintone_app_ids.json
   %LOCALAPPDATA%\AccountReconciler\kintone_api_tokens.json
   ```
2. アプリを再起動します（インストールフォルダの雛形から再読み込みされます）
3. 第3章の手順に従って再設定します

> データベースと暗号化キーには影響しません。

---

## 8-4. 担当開発者への連絡

以下の場合は担当開発者に連絡してください。

- DBマイグレーションエラーが解消しない
- ログに繰り返しエラーが記録されているが原因不明
- アプリのアップデートが必要

連絡時に以下を提供してください。

1. エラーが発生した日時
2. エラーメッセージ（画面に表示されたもの）
3. ログファイル（エラー発生日のもの）
