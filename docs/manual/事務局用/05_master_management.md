# 第5章 マスタ管理

組合員口座・銀行・委託先口座などのマスタデータを管理します。

---

## 5-1. 銀行マスタ・支店マスタ

### 画面の開き方

左側メニューから **「銀行マスタ」** をクリックします。

画面は左右に分かれており、**左列が銀行一覧**、**右列が支店一覧** です。
銀行一覧で銀行を選択すると、その銀行の支店が右列に表示されます。

### 銀行を追加する

1. 銀行一覧の **「新規追加」** ボタンをクリックします
2. ダイアログで以下を入力します

   | 項目 | 説明 |
   |------|------|
   | 銀行コード | 4桁の銀行コード（例：`0001`） |
   | 銀行名 | 半角カナで入力（例：`ﾐﾂﾋﾞｼUFJｷﾞﾝｺｳ`） |

3. **「登録」** をクリックします

### 銀行を編集・削除する

- 各行の **「編集」** ボタン：銀行コード・銀行名を修正できます
- 各行の **「削除」** ボタン：銀行を削除します（紐付く支店も削除されます）

> 削除確認ダイアログが表示されます。内容を確認してから実行してください。

### 支店を追加する

1. 銀行一覧から対象銀行を選択します
2. 支店一覧の **「新規追加」** ボタンをクリックします
3. ダイアログで以下を入力します

   | 項目 | 説明 |
   |------|------|
   | 支店コード | 3桁の支店コード（例：`001`） |
   | 支店名 | 半角カナで入力（例：`ﾎﾝﾃﾞﾝ`） |

4. **「登録」** をクリックします

### 支店を編集・削除する

- 各行の **「編集」** ボタン：支店コード・支店名を修正できます
- 各行の **「削除」** ボタン：支店を削除します

> **全銀ファイルに出力される銀行名・支店名は半角カナである必要があります。**
> 全角や漢字は使用しないでください。

---

## 5-2. 委託先口座マスタ

### 画面の開き方

左側メニューから **「委託先口座マスタ」** をクリックします。

### 委託先口座を追加する

1. **「新規追加」** ボタンをクリックします
2. ダイアログで以下を入力します

   | 項目 | 説明 |
   |------|------|
   | 表示名 | システム内での識別名（例：「メイン口座」） |
   | 委託者コード | 全銀ヘッダに使用するコード（10桁） |
   | 委託者名 | 全銀ヘッダに使用する名称（半角カナ、40文字以内） |
   | 銀行コード | 振替元銀行の4桁コード |
   | 支店コード | 振替元支店の3桁コード |
   | 口座種別 | 普通・当座など |
   | 口座番号 | 7桁の口座番号 |

3. **「登録」** をクリックします

### 有効・無効を切り替える

一覧の **「有効/無効」** ボタンで状態を切り替えます。

- **有効**：締め処理で選択できる状態
- **無効**：締め処理での選択から除外される

> 複数の委託先口座がある場合、締め処理時に使用する口座を選択できます。
> 使用しない口座は「無効」にしておくことをお勧めします。

### 委託先口座を編集・削除する

- 各行の **「編集」** ボタン：内容を修正できます
- 各行の **「削除」** ボタン：削除します（組合員口座に紐付いている場合は削除できません）

## 5-3. 組合員口座マスタ

### 画面の開き方

左側メニューから **「組合員口座マスタ」** をクリックします。

### 口座状態の意味

組合員口座には以下の4つの状態があります。

| 状態 | 説明 |
|------|------|
| **新規** | 初回登録された状態。次回振替成功時に「有効」へ遷移する。全銀ファイルの「新規コード」が「1」になります |
| **有効** | 通常の活性状態。振替成功済み |
| **変更** | 口座情報（銀行・支店・口座番号・口座名義）を変更した状態。次回振替成功時に「有効」へ遷移する。全銀ファイルの「新規コード」が「2」になります |
| **無効** | 使用停止状態。振替対象から除外されます |

> 全銀フォーマットの「新規コード」は銀行への通知区分です。
> - 新規（コード1）：初めて振替する口座
> - 変更（コード2）：前回から口座情報が変わった口座
> - 継続（コード0）：変更なし

### フィルター・検索

複数の条件で絞り込みができます。

| フィルター | 内容 |
|----------|------|
| 銀行 | 特定の銀行で絞り込み |
| 支店 | 特定の支店で絞り込み |
| 委託先 | 特定の委託先口座で絞り込み |
| 口座状態 | 新規・有効・変更・無効で絞り込み |
| キーワード | KintoneID・口座番号・口座名義の一部で検索 |

「絞込実行」で適用、「クリア」でリセットします。

### 組合員口座を新規登録する

1. **「新規追加」** ボタンをクリックします
2. ダイアログで以下を入力します

   | 項目 | 説明 |
   |------|------|
   | Kintone ユーザーID | KintoneのユーザーID（必須） |
   | 銀行コード | 4桁の銀行コード |
   | 支店コード | 3桁の支店コード |
   | 口座種別 | 普通・当座など |
   | 口座番号 | 7桁の口座番号 |
   | 口座名義 | 半角カナで入力 |
   | 委託先口座 | 使用する委託先口座を選択 |

3. **「登録」** をクリックします
4. 登録後、状態は「**新規**」になります

### 口座情報を変更する

1. 変更したい行の **「編集」** ボタンをクリックします
2. 変更内容を入力して **「保存」** をクリックします
3. 口座情報（銀行コード・支店コード・口座番号・口座名義）を変更すると、状態が **「変更」** に遷移します

> 「変更」状態の口座は、次回の締め処理で全銀ファイルに「変更コード（2）」が付与されます。
> 振替が成功すると自動的に「有効」状態に戻ります。

### 口座を無効化する

退会や口座廃止などで振替が不要になった場合に使用します。

1. 対象行の **「無効化」** ボタン（または「有効/無効」トグル）をクリックします
2. 確認ダイアログが表示されます。内容を確認して **「実行」** をクリックします
3. 状態が **「無効」** になり、以降の振替対象から除外されます

### 無効化した口座を再有効化する

1. 対象行の **「再有効化」** ボタンをクリックします
2. 確認ダイアログが表示されます。内容を確認して **「実行」** をクリックします
3. 状態が **「新規」** に戻ります（次回振替成功後に「有効」へ遷移）

### 口座を削除する

口座情報を完全に削除します。通常は「無効化」の使用を推奨します。

1. 対象行の **「削除」** ボタンをクリックします
2. 確認ダイアログが表示されます。内容を確認して **「実行」** をクリックします

> **削除すると過去の処理履歴との紐付けも失われます。**
> 退会・廃止の場合は「無効化」を使用し、削除は慎重に行ってください。
