# 第7章 バックアップ・PC移行

---

## 7-1. バックアップの重要性

このシステムのデータは以下の2つで構成されています。

| データ | 場所 | 説明 |
|--------|------|------|
| データベース | `%LOCALAPPDATA%\UnionDuesManager\SecureBank.db` | 口座情報・処理履歴・マスタデータ |
| 暗号化キー | `%APPDATA%\AccountReconciler\secret.key` | 口座情報の復号に必要 |

**この2つはセットで管理する必要があります。**
データベースだけ、またはキーだけを個別にコピーしても、別のPC・環境では口座情報を復号できません。

アプリのバックアップ機能はこの2つをZIPにまとめて保存・復元します。

---

## 7-2. バックアップ手順

### 実施タイミング（推奨）

- **毎月の締め処理前**（最重要）
- PC交換・OSアップデートの前
- 大量のマスタデータを変更した後

### 手順

1. アプリを起動し、左側メニューから **「メンテナンス」** を開きます
2. **「バックアップ」** セクションの **「バックアップ作成」** ボタンをクリックします
3. 保存先フォルダを選択します
4. **「保存」** をクリックします

バックアップファイルはZIP形式で保存されます。

```
ファイル名例：backup_UnionDuesManager_20250301_143022.zip
```

### バックアップZIPの内容

| ファイル | 説明 |
|---------|------|
| `SecureBank.db` | データベース本体 |
| `encryption_key.bin` | 暗号化キー（DPAPIの保護を外した状態で格納） |

> **バックアップZIPには暗号化キーが平文で含まれます。**
> 第三者に渡さないよう、安全な場所（外付けドライブ、パスワード保護ストレージなど）に保管してください。

### 保管場所の推奨

- 外付けHDD・USBメモリ（PCと別管理）
- 組合で管理しているクラウドストレージ（アクセス制限付き）
- 複数世代（最低3世代）のバックアップを保持する

---

## 7-3. PC載せ替え手順

PCを新しくする際の完全な手順です。

```
【旧PC】
1. バックアップを取得する（↑7-2参照）
2. バックアップZIPを外付けメディア等にコピーする

【新PC】
3. システムをインストールする（→第2章）
4. Kintone接続設定を行う（→第3章）
5. バックアップZIPをリストアする（↓7-4参照）
6. 動作確認する
```

> **「DBファイルをそのままコピーする」方法は絶対に使用しないでください。**
> 暗号化キーはWindowsのユーザーアカウント情報と結合されているため、別PCにDBを直接コピーしても口座情報を復号できません。

---

## 7-4. リストア手順

バックアップZIPからデータを復元します。

### 実施前の確認

- 新PCにアプリがインストールされていること
- バックアップZIPファイルがアクセスできる場所にあること

### 手順

1. アプリを起動し、左側メニューから **「メンテナンス」** を開きます
2. **「リストア」** セクションの **「バックアップから復元」** ボタンをクリックします
3. ファイル選択ダイアログで、バックアップZIPを選択します
4. **「開く」** をクリックします
5. 確認ダイアログが表示されます。内容を確認して **「実行」** をクリックします
6. リストアが完了すると、**アプリが自動的に再起動** します

### リストア後の処理（自動）

リストア時に以下が自動実行されます。

1. バックアップZIPからDBファイルと暗号化キーを展開
2. 暗号化キーを**新PCのDPAPI**で再保護（新PCのユーザーアカウントに紐付け直し）
3. DBファイルを所定の場所に配置
4. アプリ再起動後にDBマイグレーションを自動適用

### 動作確認

再起動後、以下を確認します。

- [ ] ダッシュボードが正常に表示される
- [ ] 「組合員口座マスタ」を開き、登録済みの口座情報が表示される（口座番号がマスクされた状態で表示されれば復号成功）
- [ ] 「締め処理履歴」で過去の処理ログが表示される

---

## 7-5. バックアップなしでの復旧（最終手段）

バックアップZIPが存在しない状態でDBや暗号化キーが失われた場合の対応です。

| 状況 | 対処 |
|------|------|
| DBのみ消失（暗号化キーは残存） | バックアップなし → マスタデータの再登録が必要 |
| 暗号化キーのみ消失（DBは残存） | 口座情報の復号不可 → 口座情報の再登録が必要 |
| 両方消失 | 全データの再登録が必要 |

いずれの場合も、銀行マスタ・委託先口座・組合員口座を再登録する必要があります。
処理ログ（締め処理・消込履歴）は復元できません。
