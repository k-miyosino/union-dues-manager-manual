# 第2章 インストール・初期セットアップ

---

## 2-1. 動作要件

| 項目 | 要件 |
|------|------|
| OS | Windows 10 バージョン 19041（2020年5月更新）以降 |
| アーキテクチャ | x64（推奨）、x86、ARM64 |
| .NET ランタイム | .NET 10.0（インストーラーに同梱、または別途インストール） |
| インターネット接続 | Kintone APIへのアクセスに必要（HTTPS） |
| ディスク空き容量 | 200MB 以上（ログ・DBの増加分を含め余裕を持って） |

---

## 2-2. インストール手順

最新のリリースは GitHub の Releases ページからダウンロードします。

### 初回のみ：署名証明書のインストール

MSIX を実行する前に、一度だけ署名証明書をインストールします。

1. `UnionDuesCollectionManager_signing.cer` をダウンロードします
2. ファイルをダブルクリック →「証明書のインストール」をクリック
3. 「ローカルコンピューター」を選択 →「次へ」
4. 「証明書をすべて次のストアに配置する」を選択 →「参照」
5. **「信頼されたルート証明機関」** を選択 →「OK」→「次へ」→「完了」

### MSIX のインストール

1. GitHub Releases ページから `.msix` ファイルをダウンロードします
2. ダウンロードした `.msix` をダブルクリックして実行します
3. Windows App Installer が起動したら「インストール」をクリックします
4. スタートメニューまたはデスクトップのショートカットからアプリを起動します

---

## 2-3. 初回起動時に自動で行われる処理

アプリを初めて起動すると、以下が自動的に実行されます。操作は不要です。

### 暗号化キーの生成

- 場所：`%APPDATA%\AccountReconciler\secret.key`
- 32バイトのランダムキー（AES-256用）が生成されます
- キーは Windows DPAPI でそのPC・ユーザーアカウントに紐付けて保護されます
- **このファイルを削除すると、登録済みの口座情報が復号できなくなります**

### データベースの作成

- 場所：`%LOCALAPPDATA%\UnionDuesCollectionManager\SecureBank.db`
- 必要なテーブルがすべて自動作成されます
- 既存のDBがある場合はマイグレーション（スキーマ更新）が自動適用されます

## 2-4. 初回セットアップ手順

初回起動後、以下の順番でセットアップを行います。

```
1. Kintone接続設定（アプリID・APIトークン）
   ↓
2. アプリ内でKintoneドメインを設定
   ↓
3. ログイン確認
   ↓
4. 銀行マスタ・委託先口座・組合員口座の登録
   （→ 事務局と連携して実施）
```

### ステップ1：Kintone接続設定

→ [第3章 Kintone接続設定](03_kintone_config.md) を参照してください。

### ステップ2：アプリ内でKintoneドメインを設定

1. アプリを起動し、ログイン画面が表示されたら一度スキップ（またはキャンセル）します
2. 左側メニューから **「設定」** を開きます
3. **「Kintoneドメイン」** 欄に組織のKintoneドメインを入力します
   - 例：`yourcompany.cybozu.com`（`https://` は不要）
4. **「保存」** をクリックします

### ステップ3：ログイン確認

1. ログイン画面でKintoneのログインIDとパスワードを入力します
2. ログインに成功し、ダッシュボードが表示されれば接続確認完了です

## 2-5. 自動更新の設定

アプリは起動時に新しいバージョンを自動で確認します。更新が検出されると、インストールを促すダイアログが表示されます。

### GitHub PAT の設定（初回のみ）

自動更新のダウンロードには GitHub Personal Access Token（PAT）が必要です。

**PAT の作成手順：**

1. GitHub にログインし、右上のアイコン →「Settings」→「Developer settings」→「Personal access tokens」→「Fine-grained tokens」を開きます
2. 「Generate new token」をクリックします
3. 設定：
   - **Repository access**：このリポジトリのみ
   - **Permissions** → **Contents**：`Read-only`
4. 「Generate token」をクリックし、表示されたトークンをコピーします

**PAT の配置手順：**

1. 以下のフォルダに `update_config.json` ファイルを作成します：
   ```
   %LOCALAPPDATA%\AccountReconciler\update_config.json
   ```
2. ファイルの内容：
   ```json
   {
     "githubPat": "github_pat_ここにトークンを貼り付ける"
   }
   ```

> PAT を設定しない場合、更新チェック自体がスキップされます（プライベートリポジトリのため認証が必要）。

### 自動更新の動作

1. アプリ起動時にバックグラウンドで最新バージョンを確認します
2. 新しいバージョンが見つかると確認ダイアログが表示されます
3. 「今すぐ更新」→ MSIX がダウンロードされます
4. 「インストール」→ アプリが終了し、Windows App Installer が起動します
5. 「インストール」をクリックすると更新が完了します
6. アプリを再起動して新しいバージョンが適用されます

> データベースや設定ファイルはアプリフォルダ外に保存されているため、アップデートでデータが消えることはありません。

### DBマイグレーション（PC載せ替え後の注意）

PC を載せ替えた後（バックアップ→リストア済み）に旧バージョンのDBを復元した場合、起動時に自動でマイグレーションが適用されます。

- マイグレーションは起動時に自動実行されるため、手動操作は不要です
- マイグレーション失敗時はログ（`%LOCALAPPDATA%\UnionDuesCollectionManager\logs\`）を確認してください
- 失敗する場合は担当開発者に連絡してください
